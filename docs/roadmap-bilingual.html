設計原則 / Design principles · 為什麼這樣設計

平台目標是因應不同市場策略，快速迭代出各種 PMI。指數開發週期越短，能探索的策略空間越大——下列四項因此是硬需求，每項都對得上既有機制。 The platform exists to iterate many PMIs quickly as market strategies change. The four pillars below are hard requirements, each backed by a concrete mechanism.

🎛️ 高可控High control

每個 factor / weight / prompt / 選股條件都精確可指定，不被黑箱牽著走。Every factor, weight, prompt and selector is explicit — no black box.

機制 declarative YAML DSL · prompt-as-code (git + sha256) · 人寫 keyword（非 LLM 自動）· factor model registry (register / promote / stage)

🧩 高彈性High flexibility

新策略 = 新 PMI，要快、且互不牽動。A new strategy is a new PMI — fast, and isolated from the rest.

機制 north-star「新增第 N 個不改前 N–1 個」· 一源一檔 poller · 一 index 一 YAML 自動 register · 可組合 factor · formula registry (NEXT-3.x)

🎯 準確 + 確定性Accurate & reproducible

分數可信、可重算、不隨機漂——同條件結果一致。Trustworthy, re-computable, non-drifting — same inputs, same score.

機制 sha256 lineage (yaml/prompt/model) · append-only audit_evaluations · evaluation cache key 完全 reuse · deterministic stub · byte-identical 回歸 · 每 score 可追回原始合約

🗂️ 多版本共存Multi-version coexistence

改 weight 不破壞歷史可比性；compliance 看「我們用 v3，過去 6 個月一致」。Editing a weight never breaks historical comparability; old versions keep computing.

機制 SCD2 core_index_definitions (version + effective_from/to) · prompt append-only -vN · 舊版本續算 · backtest replay · diff view (SHIP-3.2)

設計決策 D1–D9 Design Decisions

#決策說明 / Note
D1 Typed multigraph Edge = type / weight / direction / confidence；不是單一純量。Edge carries type/weight/direction/confidence — not a single scalar.
D2 結構邊 vs 統計邊拆開 core_market_edges（結構，append-only）vs ts_market_correlations（統計，time-series）。
D3 三量分清 probability = 值；volume = 權重；佔比 = family 衍生；不混用。
D4 Agent 提議邊不歸類節點 LLM 可提議新邊，但不能改動節點分類。
D5 Formula registry，不用 eval() 公式走靜態 registry 解析，不動態執行字串（安全）。
D6 留在 Postgres 不上 Neo4j；pgvector + JSONB + recursive CTE 夠用。Stay in Postgres — pgvector + JSONB + recursive CTEs are sufficient at this scale.
D7 雙迴圈解耦 Loop A 維護邊（graph builder）/ Loop B scoring 只讀邊。
D8 Graceful degradation 邊表空 → 退化成現況行為，不 crash。Empty edge table degrades to current behavior gracefully.
D9 GraphRAG：借 pattern，不 adopt framework GraphRAG 模式可借鑑，但不引入整套框架依賴。
