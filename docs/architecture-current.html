② Ingest 層 / Ingest Layer — pmi-ingest (asyncio)

模組結構 Module layout

  • pollers/ — polymarket_rest, polymarket_clob, polymarket_history, kalshi_rest, kalshi_clob, metaculus_rest, mock_polymarket
  • streams/ — polymarket_ws, kalshi_ws
  • chain/ — polygon_indexer, uma_resolver, cohort, abis
All pollers share the same health-tracking contract.

Health tracking 每次 poll 都寫

  • INSERT audit_source_poll_log (immutable, per poll)
  • UPSERT audit_source_health (current state, per source)
  • 原子 _upsert_market — INSERT…ON CONFLICT (CORR-3.1)
Atomic upsert prevents duplicate market rows on re-poll.

CLI 入口 CLI entrypoints

pmi-ingest run --source <name>  ·  pmi-ingest clob (orderbook)  ·  pmi-ingest ws (trade stream)  ·  pmi-ingest chain (polygon/UMA)

All commands run inside Docker via docker compose --profile pmi (per §0 rule 4 of CLAUDE.md).
