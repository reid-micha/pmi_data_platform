核心主張 / The Pitch

Polymarket 給的是單一合約的即時價格——「川普當選的機率是 62%」。但沒有人把相關合約聚合成「Fed 升息不確定性指數」或「地緣衝突風險指數」這種主題化、可比較的數字。 Polymarket gives you per-contract prices. Nobody aggregates related contracts into a themed, comparable index — a "Fed hawkishness index" or "geopolitical risk index".

傳統上自己做一個這樣的指數，需要 2–4 週的工程 + 量化 + 部署工作。這個平台讓你用一段 YAML 宣告指數，約 10 分鐘建好，全程可追溯、可一鍵回測。 Building one traditionally takes 2–4 weeks of engineering, quant work, and deployment. This platform lets you declare an index in YAML and have it running in ~10 minutes, fully auditable and one-click backtestable.

問題與機會 / Problem & Opportunity

痛點 現狀 Status quo

  • 只有單一合約價格，無主題聚合
  • 客製指數需要 2–4 週工程
  • 無版本控制：改一次 weight 就失去歷史可比性
  • 無 backtest：改完才知道對不對
  • 無 prompt / model lineage：LLM 評估不可重現

Single-contract prices only. Custom indexes take weeks. No versioning, no backtest, no LLM audit trail.

  • 主題聚合：YAML 宣告哪些合約、用哪些 factor
  • ~10 分鐘建好可運行的指數
  • SCD2 版本控制：每次改動保留歷史
  • 一鍵 90 天 backtest（規劃中）
  • prompt sha256 + model_id 全程 lineage

Thematic aggregation. ~10 min to a running index. SCD2 versioning. 90-day backtest (planned). Full LLM lineage.

使用情境 / Use Cases

研究 / 量化 Research / Quant

把宏觀觀點（例如「Fed 不會降息」）具象化為 Fed-Hawkish 指數 YAML。跑 10 個 weight 變體的 backtest，找最能捕捉市場分歧的版本。

Translate a macro thesis into a YAML index. Run 10 weight-variant backtests to find the spec that best captures market disagreement.

內容 / 媒體 Content / Media

追地緣事件：調整 War Index 的 weight 或 prompt → 立即得到新圖表 + 成分合約清單 → 直接引用進報導。現役 polymarket-war-index 真實分數 47.11。

Track geopolitical events: tweak War Index weights or prompts, get an updated chart + constituent list, cite directly in reporting. Live score: 47.11.

風險 / 法遵 Risk / Compliance

監控供應鏈中斷：建客製 Supply-Chain-Risk 指數 → dashboard feed → 分數超閾值自動觸發 alert，並附全程稽核 lineage 供法遵查核。

Monitor supply-chain disruption: custom index → dashboard feed → threshold alert with full audit lineage for compliance review.

目標是「用最少的 LLM 成本，得到最高品質的評估」。越貴的模型只在真正需要時才啟動。Minimize LLM cost while maximizing evaluation quality — expensive models only activate when necessary.

技術地基（白話）/ Tech Foundation, Plain Words

原則：能不加基礎設施就不加。Principle: add infrastructure only when forced to.

單一 PostgreSQL + pgvector

一個資料庫存所有東西。pgvector 做語意搜尋。不拆三個 DB。
One DB for everything. pgvector for semantic search. No DB sprawl.

YAML → Pydantic 宣告式 DSL

使用者寫 YAML，平台解析成 IR，engine 只認 IR。
Users write YAML; platform parses to IR; engine only sees IR.

Python asyncio 抓資料

REST 輪詢 + WebSocket 串流 + 鏈上事件，全 Python。
REST polling + WebSocket streaming + on-chain events, all Python.

MLflow 鏡像層

Prompt / run 可視化 UI；Postgres 是真相來源，MLflow 是 productivity layer。
Prompt + run visualization. Postgres is truth; MLflow is a productivity mirror.

httpx + tenacity 輪詢重試

自動 retry + source health 監控。每次 poll 記錄成功 / 失敗 / 延遲。
Auto-retry + source health monitoring. Every poll logged.

REST API live. MCP server for AI-agent access planned for P3.
