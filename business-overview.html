問題與機會

研究員想要什麼？目前拿不到什麼？

預測市場的真正價值不是「單一合約價格」，而是「組合起來的主題訊號」。但沒有工具讓你快速組。

!現在做指數的痛點

  • Polymarket 只給單一合約價格——沒有跨合約的主題聚合
  • 自己組指數要寫程式——一個指數從定義到上線 2-4 週
  • 沒有版本控、沒有 audit trail——改完 weight 不知道之前是什麼
  • 沒有 backtest——「過去半年指數怎麼跑」答不出來
  • 沒有 prompt 版本管理——AI 評估結果不可重現

✓平台提供的

  • PMI = 預測市場指數——把多個相關合約聚合成單一可追蹤分數
  • YAML 宣告，10 分鐘建一個指數——不需 dev pipeline
  • 每個分數帶完整 lineage——版本、prompt、model、合約全可追
  • 一鍵 backtest 過去 90 天
  • prompt 跟模型走 git + DB 版本控——同條件可完全重現
★ 平台核心能力 ★

PMI LIFECYCLE

一個指數從想法到上線的 5 個階段

每階段都是分鐘級。整條 pipeline 設計成「沒有人類等待人類」——研究員不需等開發、開發不需等 ops。

1
定義
~ 5 分鐘
YAML 宣告 factors、weights、合約篩選條件、prompt 引用
2
Dry-run 預覽
~ 30 秒
不寫 DB，本地跑一次完整 pipeline 看結果跟 component breakdown
3
Backtest
~ 1 分鐘
用既有歷史合約 + cached evaluation 跑 90 天回測
4
Diff 比對
~ 10 秒
跟前一版比：哪些合約進 / 出、哪些分數變了、整體分數差多少
5
Commit + Publish
~ 1 分鐘
SCD2 版本號自動 bump、舊版 retire、新版開始接 production tick
USE CASES（情境示範，非客戶承諾）

「快速迭代 PMI」的三個典型場景

同樣的能力，三組不同使用者用不同方式。情境用來測試平台 capabilities 是否夠用。

研究 / Quant

把宏觀觀點變成可追蹤訊號

研究員有觀點：「下季 Fed 不會降息」。傳統上要 quant 寫程式抓相關合約 → backtest → deploy。在平台上：5 分鐘建 Fed-Hawkish 指數 YAML，立刻看 90 天 backtest，調 weight 試 10 種版本。

需要的平台能力：YAML DSL、selector（keyword + semantic）、backtest CLI、diff CLI
內容 / 媒體

追蹤事件演變的指數

分析師追蹤地緣事件。「俄烏戰況有變」→ 改 War Index 的 weight 跟 prompt → 立刻有新的圖表跟 component 列表 → 寫報導引用「我們的 v3 指數」。

需要的平台能力：prompt versioning、SCD2 index_definition、explain endpoint、public read API
監控 / 風控

用市場價格當風險訊號

風控長要監控供應鏈中斷風險。客製 Supply Chain Risk Index → 串進內部 dashboard → 超過閾值自動 alert。需要 24/7 可靠 + 完整 audit。

需要的平台能力：schedule/cron、webhook/alert、source health monitoring、append-only audit
AI STRATEGY

四層 AI 架構：把成本跟品質分開思考

不是所有合約都需要 GPT-4o。便宜的篩選 + 便宜的評分 + 必要時才升級到深評。 這層架構讓平台可以擴展到 10k+ 合約而不爆預算。

P1 trigger logic
為什麼分層： Polymarket 活躍合約 ~1000-3000，全部跑 Tier 1 一次 ~$1，每天 12 tick = $12/天。 如果全部跑 Tier 2 (Sonnet) = ~$50/天，10× 成本。Tier 階梯讓平台可隨意加新指數而不會成本爆炸。
TECH STACK

技術選型對齊「快速迭代」

每個元件選擇的理由：要不要讓「新增 N+1 個東西」不需要改前 N 個東西。

Schema 演化
PostgreSQL + Alembic
SCD Type 2 給 index_definition；append-only audit；prompt 用 sha256 cache key。一個 DB 不分家，dev/prod 對齊。
指數定義語言
YAML → Pydantic IR
user 寫 YAML，engine 拿 IR。沒寫程式 = 沒 deploy 風險。schema 變更可向後相容。
Pipeline 引擎
Python asyncio + SQLAlchemy 2.0
純函式式 selector → resolver → evaluator → aggregator。每段可獨立測 + 替換。
Model / Prompt 版本控
MLflow (mirror)
DB 是 source of truth，MLflow 是 UI。MLflow 掛了 pipeline 不停。可換 Weights & Biases 不重做。
資料擷取
httpx + tenacity + structlog
async REST polling 已上，WS / chain indexer 是同 pattern 加 poller 檔案。新資料源 = drop in。
外部介面
FastAPI + MCP server
REST 給 web / 自動化，MCP 給 AI agent。同一個資料層雙協議暴露。
背景任務
Arq → Temporal（演進）
MVP 用 Arq 簡單，長 workflow / backtest 痛了再加 Temporal。不 over-engineer。
觀測
structlog → Grafana Cloud
JSON 結構化 log + 三個基礎面板（source health / LLM cost / PMI freshness）。Sentry 抓錯。
